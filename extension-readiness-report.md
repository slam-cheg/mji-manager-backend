# Отчет о готовности расширения к новому workflow

## Требуемый workflow:

1. ✅ Из расширения передается файл на анализ и адрес объекта
2. ❌ Найти на какой странице файла начинается отчет с этим объектом
3. ❌ Передать 10 страниц (включая найденную) в DeepSeek
4. ❌ Получить ответ, взять эти данные и отправить в DeepSeek снова с другим запросом - минимально перефразировать комментарии к дефектам
5. ❌ Вернуть перефразированный JSON в расширение
6. ❌ Расширение сделает вставку полей в соответствующие инпуты на сайте

**Дополнительно:**
- ❌ При включенном AI режиме функция вставки из localStorage должна сначала через deepseek прогонять вставляемый текст для минимального перефразирования

---

## Текущее состояние:

### ✅ Что уже есть:

1. **Отправка файла на сервер** (`parserPDF.ts`):
   - Файл отправляется через `UPLOAD_PDF` message
   - Передается `useAI` флаг
   - Передается `prevSurveyNumber` (номер предыдущего обследования)
   - Файл конвертируется в base64

2. **AI режим переключатель** (`createPopup.ts`):
   - Есть переключатель `#useAI`
   - Состояние сохраняется в `window.appVariables.aiswicherState`
   - Переключатель отображается только если `window.appData.functions.useAI === true`

3. **Обработка ответа от сервера** (`messageUtils.ts`):
   - Данные сохраняются в `localStorage.setItem("parsedPDF", ...)`
   - Лоадер скрывается после получения ответа

4. **Вставка данных из localStorage** (`loadData.ts`):
   - Есть функция `loadData()` которая вставляет данные в поля формы
   - Данные читаются из `localStorage.getItem("MJIDATA")`
   - Вставка происходит напрямую без AI обработки

5. **Адрес объекта** (`searchAllInputs.ts`):
   - Адрес извлекается: `window.appVariables.address = window.appVariables.htmlBody.querySelector("#comboboxTextcomp_12339").value`

---

### ❌ Что отсутствует:

#### 1. Отправка адреса объекта на сервер
**Текущее состояние:**
- В `parserPDF.ts` отправляется только `prevSurveyNumber`, но не адрес объекта
- Адрес доступен в `window.appVariables.address`, но не передается

**Что нужно:**
- Добавить `address: window.appVariables.address` в payload при отправке `UPLOAD_PDF`

#### 2. Поиск страницы с адресом объекта в PDF
**Текущее состояние:**
- ❌ Полностью отсутствует
- Весь файл отправляется на сервер без поиска нужной страницы

**Что нужно:**
- На бэкенде: функция поиска страницы в PDF, где начинается отчет с указанным адресом
- Можно использовать DeepSeek для поиска адреса на страницах (по 10-20 страниц за раз)
- Или использовать Python парсер для быстрого поиска по тексту

#### 3. Отправка только 10 страниц в DeepSeek
**Текущее состояние:**
- В `deepseek-parser.service.ts` есть метод `extractPdfPages()` для извлечения страниц
- Но нет логики поиска нужной страницы перед извлечением

**Что нужно:**
- После нахождения страницы с адресом, извлечь 10 страниц начиная с найденной
- Передать эти страницы в DeepSeek для парсинга

#### 4. Второй этап - перефразирование комментариев
**Текущее состояние:**
- В `upload.service.ts` есть метод `modifyDataWithAI()`, но он заглушка
- Нет логики перефразирования только комментариев к дефектам

**Что нужно:**
- После получения результата парсинга, извлечь все поля с комментариями к дефектам
- Отправить их в DeepSeek с промптом на минимальное перефразирование
- Вернуть перефразированный JSON

#### 5. Вставка данных из parsedPDF в форму
**Текущее состояние:**
- Данные сохраняются в `localStorage.setItem("parsedPDF", ...)`
- Но нет логики чтения этих данных и вставки в форму
- `loadData()` читает только из `localStorage.getItem("MJIDATA")`

**Что нужно:**
- Добавить логику чтения из `localStorage.getItem("parsedPDF")`
- Преобразовать структуру данных из DeepSeek в формат, который понимает `loadData()`
- Вызвать вставку данных в поля формы

#### 6. AI режим при вставке из localStorage
**Текущее состояние:**
- В `loadData()` нет проверки `window.appVariables.aiswicherState`
- Данные вставляются напрямую без AI обработки

**Что нужно:**
- Проверить `window.appVariables.aiswicherState` перед вставкой
- Если AI включен, отправить данные в DeepSeek для перефразирования
- После получения перефразированных данных, вставить их в форму

---

## Структура данных:

### Текущий формат (из `loadData.ts`):
```typescript
loadData["Результаты выборочного обследования"]["Крыша"][window.appVariables.krovlyaName]["Выявленные дефекты"]
```

### Формат от DeepSeek (из `deepseek-parser-result.json`):
```json
{
  "результаты_обследования": [
    {
      "перечень_и_характеристика_элементов": "...",
      "характер_и_местоположение_выявленных_дефектов": "...",
      "оценка_по_предыдущему_обследованию": "...",
      "процент_дефектной_части": 5,
      "оценка": "У"
    }
  ]
}
```

**Проблема:** Структура данных от DeepSeek не соответствует ожидаемой структуре в `loadData()`

**Решение:** Нужна функция преобразования структуры DeepSeek → структура для вставки
