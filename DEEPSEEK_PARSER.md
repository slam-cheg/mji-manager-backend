# Парсинг PDF через DeepSeek API

## Описание

Реализован альтернативный метод парсинга PDF документов через DeepSeek API. Этот метод позволяет обрабатывать большие документы (до 4000+ страниц, до 100 МБ) с разбивкой на части. Поддерживает как текстовые PDF (через pdf-text), так и отсканированные документы (через mistral-ocr).

## Возможности

✅ **Поддержка больших документов:**
- До 4000+ страниц
- Файлы до 100 МБ
- Автоматическая разбивка на части по 10 страниц

✅ **Поддержка разных типов PDF:**
- **Текстовые PDF:** Используется `pdf-text` для прямого извлечения текста (быстрее и точнее)
- **Отсканированные PDF:** Используется `mistral-ocr` для распознавания текста из изображений

✅ **Структурированный вывод:**
- Данные извлекаются в формате `template.json`
- Автоматическое объединение результатов из разных частей документа

## Настройка

### Переменные окружения

Добавьте в `.env` файл:

```env
ROUTERAI_API_KEY
ROUTERAI_API_URL
ROUTERAI_DEEPSEEK_MODEL
```

**Автоматическое определение движка:**
- Парсер автоматически анализирует первые страницы PDF
- Если извлекается достаточно текста (>500 символов) → используется `pdf-text` (текстовый PDF)
- Если текста мало → используется `mistral-ocr` (отсканированный PDF)
- Не нужно указывать движок вручную!

### Лимиты

- **Лимит токенов за запрос:** 131000 токенов
- **Страниц за запрос:** 10 страниц (настраивается через `PAGES_PER_CHUNK`)
- **Таймаут запроса:** 5 минут

## Использование

### Через API

Отправьте POST запрос на `/api/upload-pdf` с параметром `useDeepSeek: true`:

```json
{
  "fileName": "document.pdf",
  "fileData": "data:application/pdf;base64,...",
  "useAI": false,
  "prevSurveyNumber": "",
  "useDeepSeek": true
}
```

### Программно

```typescript
import { DeepSeekParserService } from './appData/deepseek-parser.service';

// В вашем сервисе
const reports = await this.deepSeekParser.parsePdfWithDeepSeek(filePath);
```

## Как это работает

1. **Определение количества страниц:**
   - Используется Python скрипт с `pdfplumber` или `PyPDF2`
   - Определяется общее количество страниц в документе

2. **Разбивка на части:**
   - Документ разбивается на части по 10 страниц
   - Каждая часть обрабатывается отдельно

3. **Автоматическое определение типа PDF:**
   - Парсер анализирует первые 3 страницы PDF
   - Если извлекается достаточно текста (>500 символов) → используется `pdf-text`
   - Если текста мало → используется `mistral-ocr`
   - Движок определяется автоматически, не нужно указывать вручную

4. **Отправка в DeepSeek:**
   - Каждая часть отправляется в DeepSeek API с указанием диапазона страниц
   - Используется плагин `file-parser` с автоматически определенным движком
   - Параметр `pages: "1-10"` указывает диапазон страниц

5. **Извлечение данных:**
   - DeepSeek анализирует страницы и извлекает данные согласно `template.json`
   - Возвращает структурированный JSON

6. **Объединение результатов:**
   - Результаты из разных частей объединяются
   - Отчеты с одинаковым регистрационным номером объединяются в один
   - Пустые значения не перезаписывают существующие данные

## Преимущества перед Python парсером

✅ **Работа с отсканированными документами** - OCR распознает текст из изображений (при использовании mistral-ocr)
✅ **Лучшая точность** - AI понимает контекст и структуру документа
✅ **Гибкость** - Легко адаптируется к изменениям в формате документа
✅ **Обработка больших файлов** - Автоматическая разбивка на части
✅ **Быстрая обработка текстовых PDF** - pdf-text быстрее и точнее для текстовых документов

## Недостатки

❌ **Требует API ключ** - Нужен доступ к OpenRouter API
❌ **Медленнее** - Каждая часть обрабатывается отдельно (1 секунда задержка между запросами)
❌ **Стоимость** - Использование API платное (зависит от тарифа OpenRouter)
❌ **Зависимость от интернета** - Требуется стабильное соединение

## Рекомендации

- Используйте DeepSeek парсер для:
  - Отсканированных документов (с `mistral-ocr`)
  - Документов с нестандартным форматированием
  - Больших документов (100+ страниц)
  - Когда Python парсер не справляется
  - Текстовых PDF (с `pdf-text` - быстрее и точнее)

- Используйте Python парсер для:
  - Текстовых PDF со стандартным форматированием
  - Когда нужна максимальная скорость без API вызовов
  - Когда нет доступа к API
  - Офлайн обработки

## Отладка

Если возникают проблемы:

1. Проверьте переменные окружения
2. Убедитесь, что API ключ валиден
3. Проверьте логи в консоли - там будет информация о каждой части
4. Убедитесь, что файл не поврежден и доступен для чтения

## Технические детали

- **Модель:** `deepseek/deepseek-v3.2` (настраивается через `ROUTERAI_DEEPSEEK_MODEL`)
- **PDF движок:** 
  - Определяется автоматически на основе анализа PDF
  - `pdf-text` - для текстовых PDF (быстрее и точнее)
  - `mistral-ocr` - для отсканированных PDF (требует OCR)
  - Не нужно указывать вручную!
- **Размер чанка:** 
  - 10 страниц для обычных файлов
  - 5 страниц для файлов > 1000 страниц (автоматически)
- **Максимум токенов:** 131000 (настраивается через `MAX_TOKENS`)
- **Задержка между запросами:** 1 секунда
- **Retry логика:** До 3 попыток при ошибках
